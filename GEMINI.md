# Instructions pour Gemini CLI

Ces instructions s'appliquent à toutes les interactions avec ce projet.

## Mode de fonctionnement

- **Autorisation implicite** : L'utilisateur accepte d'avance toutes les modifications de code, créations de fichiers et exécutions de commandes shell nécessaires à la réalisation des tâches demandées. Ne demandez pas d'autorisation supplémentaire.
- **Narration minimale** : Supprimez les préambules et les explications détaillées avant d'agir. Foncez directement à l'exécution dès que l'objectif est compris.
- **Autonomie maximale** : Prenez les décisions techniques nécessaires pour respecter les standards du projet sans solliciter l'utilisateur, sauf en cas d'ambiguïté majeure sur le besoin métier.
- **Pas de confirmation** : Ne demandez pas confirmation avant de lancer des outils (shell, replace, etc.). L'interface de l'agent gère déjà les aspects de validation si nécessaire.

## Standards Techniques

- Suivre les conventions existantes dans le projet (React, Next.js, Prisma, Tailwind).
- Prioriser la lisibilité et le contraste (texte noir sur fond clair).
